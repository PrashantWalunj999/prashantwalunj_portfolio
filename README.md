# Prashant Walunj — Portfolio Website

A modern, animated personal portfolio website built with Python/Flask for local development, compiled to a static site for GitHub Pages hosting, and containerised with Docker for consistent deployments.

---

## Project Structure

```
prashantwalunj_portfolio/
│
├── app.py                        # Flask dev server entry point
├── build.py                      # Generates static site into docs/
├── requirements.txt              # Python dependencies (Flask)
├── Dockerfile                    # Multi-stage Docker image (builder + runtime)
│
├── templates/
│   └── index.html                # Jinja2 template — single source of truth for all content
│
├── static/
│   ├── css/
│   │   └── style.css             # All styling — themes, animations, responsive breakpoints
│   ├── js/
│   │   └── main.js               # Dark mode, typewriter, scroll reveal, ripple, counters
│   ├── images/
│   │   └── prashant.jpg          # Profile photo
│   └── files/
│       └── resume.pdf            # Downloadable CV
│
├── prashant/                     # Source assets (not served directly)
│   ├── PrashantWalunj_image.jpg
│   └── PrashantWalunj_Resume.pdf
│
├── docs/                         # GitHub Pages output — generated by build.py (do not edit)
│   ├── index.html
│   ├── .nojekyll
│   └── static/                   # Copied from static/ at build time
│
└── devops/                       # CI/CD and container configuration
    ├── docker-compose.yml        # Local dev environment (up/down)
    ├── azure-pipelines.yml       # Azure DevOps — build & push to ACR
    └── github-actions.yml        # GitHub Actions — build & push to ACR
```

## Features

| Feature | Details |
|---|---|
| Dark mode | Toggle with persistence via `localStorage` |
| Typing animation | Cycles: DevOps Engineer / DevSecOps Engineer / Cloud Engineer |
| Orbital skill badges | CSS rotate-translate-counterrotate animation |
| Scroll reveal | `IntersectionObserver` staggered fade-up on cards |
| Animated counters | Stat numbers count up on scroll into view |
| Certifications | Hover accent border + glow per cert type (Terraform / Scrum / Microsoft) |
| Contact form | Mailto handler with pre-filled subject and body |
| About modal | Overlay popup triggered from nav |
| Resume download | Direct PDF download button |
| Back to top | Appears after 400px scroll |
| Responsive | 3 breakpoints: 900px / 600px / 380px |
| OG / Twitter meta | Social preview image and description |

## Run Locally — Flask (no Docker)

```bash
# 1. Install dependencies
pip install -r requirements.txt

# 2. Start dev server
python app.py
```

Open `http://127.0.0.1:5000`

---

## Run Locally — Docker

The `docker-compose.yml` is inside `devops/` and references the `Dockerfile` at the project root.

```bash
# From the devops/ folder:
cd devops

# Build image and start container
docker compose up --build

# Stop and remove container
docker compose down
```

Or from the project root:

```bash
docker compose -f devops/docker-compose.yml up --build
docker compose -f devops/docker-compose.yml down
```

Open `http://localhost:5000`

> **Live reload:** `templates/` and `static/` are bind-mounted as read-only volumes, so CSS/HTML edits reflect on refresh without rebuilding the image.

## Build Static Site for GitHub Pages

```bash
# From project root
python build.py
```

This renders the Flask template to plain HTML, rewrites all `/static/` paths to relative `static/` paths, copies assets, and outputs everything into `docs/`.

```bash
git add docs/
git commit -m "Build static site"
git push
```

### Enable GitHub Pages

1. Go to repository **Settings > Pages**
2. Source: `Deploy from branch` → Branch: `main` → Folder: `/docs`
3. Save

Live at: `https://<username>.github.io/<repo-name>/`

## CI/CD — Build & Push Docker Image to Azure Container Registry

### Azure DevOps (`devops/azure-pipelines.yml`)

**Required pipeline variables** (set in Azure DevOps → Pipelines → Variables):

| Variable | Example |
|---|---|
| `ACR_NAME` | `myregistry` |
| `ACR_SERVICE_CONN` | `my-acr-service-connection` |
| `IMAGE_NAME` | `prashantwalunj-portfolio` |

Triggers on push to `main` (excluding `docs/` and `README.md`). Builds the Docker image, pushes with `build-<id>` and `latest` tags, then runs a smoke test pulling the image and verifying an HTTP 200 response.

### GitHub Actions (`devops/github-actions.yml`)

**Required repository secrets** (Settings → Secrets and variables → Actions):

| Secret | Example |
|---|---|
| `ACR_LOGIN_SERVER` | `myregistry.azurecr.io` |
| `ACR_USERNAME` | ACR admin / service principal |
| `ACR_PASSWORD` | ACR password / SP secret |
| `IMAGE_NAME` | `prashantwalunj-portfolio` |

Same trigger behaviour as the Azure DevOps pipeline. Uses Docker layer caching via GitHub Actions cache for faster subsequent builds.
